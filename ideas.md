**Design Movement**: Data Brutalism

**Core Principles**:
1. Raw authenticity - expose the structure, celebrate data in its native form
2. Monospace typography dominance - code is the aesthetic
3. High contrast, terminal-inspired color schemes
4. Functional minimalism - every element serves a purpose

**Color Philosophy**: 
Terminal-inspired palette with stark contrasts. Deep charcoal backgrounds (#0a0e14) with vibrant accent colors borrowed from syntax highlighting - electric cyan (#39bae6), neon green (#59c9a5), warning amber (#ffb454). The intent is to evoke the feeling of a sophisticated development environment where data lives and breathes.

**Layout Paradigm**: 
Grid-based modular system with visible borders and structure lines. Asymmetric content blocks that feel like terminal windows or code editors. Sections are clearly delineated with borders, creating a dashboard-like experience where each project is a "module" that can be inspected.

**Signature Elements**:
1. Monospace fonts for all headings and data displays
2. ASCII art dividers and decorative elements
3. Syntax-highlighted code snippets as visual texture

**Interaction Philosophy**: 
Instant, snappy interactions. Hover states reveal metadata like a debugger tooltip. Clicks feel like executing commands. No unnecessary flourishes - every animation is purposeful and quick.

**Animation**: 
Terminal-style typing effects for headings on load. Quick fade-ins (100-200ms) for content blocks. Hover states use sharp color transitions without easing - instant on/off like a switch. Project cards "boot up" sequentially with a brief flash effect.

**Typography System**: 
Primary: JetBrains Mono or Fira Code for headings, labels, and data
Secondary: IBM Plex Mono for body text
Hierarchy through weight and color, not size variation - everything stays relatively uniform to maintain the terminal aesthetic
</text>

**Design Movement**: Swiss Modernism meets Data Visualization

**Core Principles**:
1. Mathematical precision in spacing and alignment
2. Information hierarchy through scale and weight
3. Grid systems as visible design elements
4. Clarity and legibility above all

**Color Philosophy**: 
Sophisticated neutral base with data-driven accent colors. Warm off-white background (#fafaf8) with deep charcoal text (#1a1a1a). Accent palette derived from data visualization best practices - distinct hues that work for colorblind users: deep blue (#2563eb), coral (#f97316), teal (#14b8a6), violet (#8b5cf6). Colors are used sparingly and meaningfully to guide attention and categorize information.

**Layout Paradigm**: 
Strict grid system with visible baseline grid. Asymmetric two-column layout where the left column is narrower (navigation/metadata) and the right expands for content. Large typography creates visual anchors. Generous whitespace creates breathing room between dense data sections.

**Signature Elements**:
1. Oversized numerical data points as visual anchors
2. Hairline rules (1px) to separate content zones
3. Circular badges for tags/skills with precise spacing

**Interaction Philosophy**: 
Smooth, considered transitions that respect user attention. Interactions feel deliberate and refined. Hover states subtly shift elements forward in space. Scrolling reveals content with parallax depth cues.

**Animation**: 
Gentle fade-up animations (300-400ms cubic-bezier easing) for content on scroll. Numbers count up when they enter viewport. Hover states use subtle scale transforms (1.02x) with soft shadows appearing. Page transitions slide content horizontally with a brief overlap.

**Typography System**: 
Primary: Inter Variable for UI elements and body text
Display: Space Grotesk for large headings and numerical data
Monospace: JetBrains Mono for code snippets only
Scale: 14px body, 18px subheadings, 32-48px section titles, 64-96px hero numbers
</text>

**Design Movement**: Neo-Glassmorphism with Data Poetry

**Core Principles**:
1. Layered depth through transparency and blur
2. Soft, organic shapes contrasting with sharp data
3. Light as a design material - gradients and glows
4. Playful sophistication - serious work presented beautifully

**Color Philosophy**: 
Dreamy gradient backgrounds with frosted glass overlays. Base gradient from deep indigo (#1e1b4b) through purple (#7c3aed) to soft pink (#ec4899). Glass cards use rgba backgrounds with 20% opacity and backdrop blur. Text is crisp white (#ffffff) with subtle glow effects. Accent colors are bright and saturated - electric blue (#3b82f6), lime green (#84cc16), hot pink (#ec4899) - used for interactive elements and data highlights.

**Layout Paradigm**: 
Floating card system with overlapping layers. Hero section uses a diagonal split with content on one side and an abstract data visualization on the other. Projects are presented as glass cards that appear to float at different depths. Sidebar navigation is a frosted glass panel that slides in. Sections flow organically rather than rigid grid alignment.

**Signature Elements**:
1. Frosted glass cards with subtle border gradients
2. Soft glow effects on hover (box-shadow with color)
3. Abstract gradient mesh backgrounds that shift on scroll

**Interaction Philosophy**: 
Fluid, delightful micro-interactions. Elements feel weightless and responsive. Hover states lift cards forward with enhanced glow. Clicks create ripple effects. The interface feels alive and responds to user presence with subtle ambient animations.

**Animation**: 
Smooth spring physics (react-spring style) for all transitions. Cards float in with staggered delays (100ms apart). Hover states scale up (1.05x) with enhanced blur and glow over 400ms. Background gradients slowly shift hue on an infinite loop (60s duration). Scroll-triggered parallax on background layers creates depth.

**Typography System**: 
Primary: Plus Jakarta Sans for headings - rounded, friendly, modern
Secondary: Inter for body text - clean and readable
Monospace: Fira Code for code snippets with ligatures
Scale: 16px body, 24px subheadings, 48-72px hero titles with gradient text fills
Letter spacing: Slightly increased (0.02em) for headings to create airiness
</text>
